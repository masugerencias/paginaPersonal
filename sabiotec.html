"SABIOTEC" desafío de tripulaciones

El Desafío de Tripulaciones es el reto en el que los alumnos de los diferentes filiales de The Bridge forman equipo y sacan adelante una Web App encargada por una empresa externa..
Es la prueba que pone el broche final al bootcamp para entrenar cómo es trabajar dentro de en un equipo tech, multidisciplinar y con tiempos y necesidades diferentes según la fase en la que esté 📲.

En el proyecto colaboran desarrolladores web, UX/UI designers, técnicos de ciberseguridad, científicos de datos y especialistas en marketing digital. Cada bootcamp aporta su conocimiento de manera que trabajan como lo harían en una empresa real y salen auténticos proyectos digitales, escalables y rentables.

Sabiotec es una app orientada al aprendizaje sobre temas tecnológicos para la población de la tercera edad. Trata de hacer menos evidente la brecha tecnológica que sufren estas personas y ayudarles a usar cosas como el email, las apps online de bancos, aprender a identificar estafas online, etc. Dentro de nuestro producto mínimo viable (PMV) el usuario podrá registrarse, iniciar sesión, elegir el tema que quiere aprender, ver un tutorial y responder a unas preguntas sobre ese tutorial.

FrontEnd
En la parte del cliente se harán las diferentes vistas por las que puede navegar un usuario. La primera página al acceder a la aplicación es el Login, donde el usuario podrá registrarse y/o iniciar sesión con sus correspondientes validaciones de seguridad. Posteriormente las páginas a las que se puede acceder son las siguientes:
"Aprendamos", "Categoría", "Home", "logros", "Pregunta, "Test" y "Tutoriales". Al iniciar sesión el usuario podrá acceder a cualquiera de estás páginas con su contenido. Paralelamente en su perfil podrá completar más detalles a modo de un registro complementario y también ver la información personal.

Backend
El back-end se encargará de guardar todos los datos del registro en la Base de datos y comprobar esos datos una vez el usuario inicia sesión. Se encargará también de recoger los datos de la BD para mostrarle al usuario sus datos y sobretodo de mostrar el contenido de cada página, es decir, el contenido de estas páginas está almacenado en MongoDB y no en la parte del cliente.

Para más detalles dejo aquí el enlace a GITHUB para poder ver el código en detalles: "Proyecto FullBack EJS PUG".
